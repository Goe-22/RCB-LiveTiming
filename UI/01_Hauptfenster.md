# RCB LiveTiming – Hauptfenster

## Zweck

Das Hauptfenster ist die zentrale Arbeitsoberfläche während des Trainings.

Von hier aus kann der Zeitnehmer alle wichtigen Informationen überwachen, ohne zwischen verschiedenen Fenstern wechseln zu müssen.

Die Zeitnahme läuft jederzeit im Hintergrund weiter.

---

# Kopfbereich

Anzeige:

- RCB LiveTiming
- Aktive Strecke
- Decoderstatus
- Homepage-Status
- Uhrzeit
- Anzahl aktiver Fahrer

Beispiel:

RCB LiveTiming

Strecke: Onroad

Decoder: 🟢 Verbunden

Homepage: 🟢 Verbunden

Fahrer aktiv: 8

---

# Haupttabelle

Spalten:

| Name | Klasse | Schnellste | Letzte | Durchschnitt | Runden | Status |

Die Tabelle wird automatisch aktualisiert.

---

# Sortierung

Die Tabelle kann sortiert werden nach:

- Name
- Klasse
- Schnellste Runde
- Letzte Runde
- Durchschnitt
- Rundenzahl

Die zuletzt gewählte Sortierung wird gespeichert.

---

# Farben

🟢 Fahrer auf der Strecke

🟡 Fahrer in der Box

🟣 Persönliche Bestzeit

🔵 Tagesbestzeit

🟠 Unbekannter Transponder

🔴 Fehler

---

# Klickfunktionen

Klick auf Name

→ Rundentabelle öffnen

Klick auf Transpondernummer

→ Fahrer zuordnen

Rechtsklick auf Fahrer

→ Kontextmenü

---

# Letzte Durchfahrt

Unter der Tabelle wird immer angezeigt:

Name

Klasse

Letzte Runde

Persönliche Bestzeit

Zeitpunkt

Beispiel:

Christian Göhler

TT-02

23.456 Sekunden

18:32:18 Uhr

---

# Statusleiste

Anzeige:

- Decoderstatus
- Homepage
- Datenbank
- Netzwerk
- Trainingsdauer

---

# Bedienprinzip

Während des Trainings darf keine Funktion die Zeitnahme unterbrechen.

Alle Änderungen erfolgen im Hintergrund.

Alle Daten werden automatisch gespeichert.
